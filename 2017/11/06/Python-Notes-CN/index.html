2.1. exec() in a scope

We will execute this in our own scope, so that we won’t influence current namespace/scope.
This is kind of namespace isolation.

If we are using scope=vars() then we can change current execution context…

2.2. eval() to compute

eval() could be executed on a specific scope, or the global scope.

4.1. list

4.1.1. Slice

  1. 1st inclusive, 2nd exclusive
  2. using step in slice, step could be negative, numbers[0:10:2] and numbers[8::-2]
  3. without step, it’s implicitly set as 1. Which means, 1st postion has too be at the left side than 2nd position.

Can use negative index is very powerful at sometimes.
Next is to generate fibonacci number sequence.

4.1.4. list object function

  1. numbers.append(10): append a value
  2. numbers.count(10)
  3. numbers.index(10): return the 1st index of 10
  4. numbers.extend([10,11]): append a list
  5. numbers.insert(3,10): insert 10 at the index of 3
  6. numbers.pop(): remove last value in the list, and return the value
  7. numbers.pop(0): remove 1st value in the list, and return the value
  8. numbers.remove(10): remove element of 10 in the list
  9. numbers.reverse()
  10. numbers.sort(): will sort the existing numbers list
  11. sorted(numbers): will return one sorted numbers, but existing one won’t change
  12. numbers.sort(reverse=True): will sort the existing numbers list, but in reverse order
  13. numbers.sort(key=len): using len() function to sort
  14. numbers.sort(cmp): using cmp(x,y) function to sort,cmp(x,y) must be defined first
  15. cmp/key/reverse can also be applied to sorted() function
  16. reversed(numbers): will return reversed numbers, but existing one won’t change
  • Note:
    1. pop() will change the list object itself and return the popped list element value.
    2. Using pop() and append() can implement the Stack data structure.

Because tuple is immutable, so it can be the key of dictionary, however, list can not.

4.3. Dictionary

Several features:

  1. Automatically added, could use misc. key types.

  2. Membership check in keys. When you do membership check using in, it checks the keys of the dictionary.
    using k in d…

4.3.1. Several functions based on dictionary

  1. clear(): clear a dict in-place.
  2. copy(): do a shallow copy of a dict.

  3. fromkeys(param:list): use the given list as the newly created dict keys’ names.

  4. get(k): will return None if key doesn’t exist. However, d[k] will give an error.

  5. items()/iteritems(): get the items of dict, iteritems() return an iterator which is more efficient and can even convert to a list. We also have keys()/iterkeys() and values()/itervalues().

  6. setdefault(k,default_v): will get the value of key k, if k not exist, set it and return with default_v value.

  7. pop(k): return with corresponding key k and remove it from dict.

  8. popitem(): return with random key and remove it from dict.
  9. update(another_dict): using another_dict to update current dict, for same key, will replace, for non-exist keys, will add.
